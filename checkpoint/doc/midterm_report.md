# 本科毕业论文中期检查报告


## 一、课题进展情况

目前本课题已经完成了问题分析、在线模拟器方案设计和主要系统实现，系统能够进行端到端测试并得到初步实验结果。前期首先围绕研究问题开展了较系统的调研工作，重点分析了基于实例负载混合指标的方法与基于请求性能预测的方法，并结合已有现象分析了传统调度方法在表达能力和参数敏感性上的局限。在此基础上，课题工作逐步收敛到实现在线模拟器进行性能预测这一核心部分，围绕如何在真实推理集群中维护可用于调度的实例状态镜像，并在请求到达时快速给出可靠预测用于请求调度展开设计与实现。

在研究主线明确之后，首先推进了在线模拟器的系统方案设计。围绕在线性能预测在全局调度流程中的语义，完成了模拟器与调度器之间的接口设计，通过 `Predict`、`Add_request` 和 `Sync_State` 三个核心接口定义预测、请求提交和状态校准三类操作。其中，`Predict` 用于预测请求到达实例后的TTFT，`Add_request` 用于在调度决策确定后更新模拟器状态，`Sync_State` 则用于接收实例执行后的状态变化并修正模拟器状态。为了保证预测结果在分布式场景下仍具可用性，重点设计了由推理实例主动上报状态变化的同步机制，并提出真实状态、演进状态和模拟状态三层状态管理结构，用于区分真实执行路径、已预测但尚未完全校准的演进路径，以及单次预测时临时使用的模拟路径，从而在保证状态隔离的同时复用前序请求的预测结果。

在方案设计完成后，工作的重点转向模拟器的工程实现。当前已经完成模拟器主体的开发，并将其接入现有 Rust 全局调度框架中用于端到端测试。为支撑模拟器状态同步，对 vLLM 推理框架进行了修改，在批处理构建与执行结束等关键位置记录批处理组成、请求状态变化等信息，并通过状态上报传递给调度器进行模拟器状态同步。与此同时，性能预测部分已经按离线剖析、在线查表的方式完成主要代码实现，能够依据模拟器构造出的批特征快速估计对应执行时长。模拟器中实现vLLM调度逻辑以预测未来状态变化和批处理结果。至此，在线模拟器已经具备了状态同步、状态演进、请求预测等关键能力。

在实验工作方面，目前已经完成实验平台搭建，并基于已实现的在线模拟器开展了一轮端到端请求重放实验。当前实验环境为 8 张 NVIDIA H20 GPU 组成的推理集群，测试模型覆盖 Dense 架构的 Qwen2.5-7B 和 MoE 架构的 Qwen3-30B-A3B，工作负载覆盖 ChatBot、Coder、Agent 三类典型场景。现阶段，模拟器已经能够在真实请求到达时为各候选实例生成预测结果，并在统一框架下支撑 Predictor 与其他基线策略的公平比较。除端到端性能对比外，目前还完成了模拟器准确性测试：在 Agent 3.1 倍负载下，Qwen2.5-7B 模型中有 84.09% 的请求预测误差控制在 30% 以内，Qwen3-30B-A3B 模型中有 75.97% 的请求预测误差控制在 30% 以内。这说明当前模拟器对 Dense 模型具有较高预测精度，在 MoE 模型上虽然误差略大，但仍能提供对调度有用的相对性能信息。整体来看，目前课题已经从方法分析阶段进入在线模拟器实现基本完成，通过测试比较调度性能的阶段。

## 二、课题研究已取得的阶段性成果

目前已经取得的阶段性成果主要集中在在线模拟器的实现进展和验证结果两个方面。在实现进展方面，当前已经形成了可运行的在线模拟器原型，并完成了其与全局调度流程的集成。模拟器已经实现 `Predict`、`Add_request` 和 `Sync_State` 三个核心接口，能够分别承担在线预测、请求确认写入和状态同步校准三类职责。围绕在线场景下最关键的状态一致性问题，当前已经完成真实状态、演进状态和模拟状态三层状态管理逻辑，实现了“预测过程不污染真实状态、已完成的预测结果可复用、状态偏移可回滚校准”的核心机制。与此同时，实例侧状态上报通路也已打通，vLLM 可以将批处理组成、请求变化和 KV Cache 变化等信息增量回传给调度器侧模拟器，使模拟器能够持续维护实例状态镜像。性能预测部分已经完成主要实现，能够在模拟器生成批特征后快速查表得到执行时长，现有测试表明其在线时延满足进入调度关键路径的要求。

在测试结果方面，当前已经完成基于该模拟器的一批端到端实验，测试模型包括 Qwen2.5-7B 和 Qwen3-30B-A3B，工作负载包括 ChatBot、Coder、Agent 和 ToolAgent。已有结果表明，由模拟器驱动的 Predictor 策略能够显著改善 TTFT：在 Qwen ChatBot 负载、22 QPS 下，相比 Company-X，TTFT Mean 改善 31.0%，TTFT P95 改善 36.3%；在 Qwen Coder 负载、7.5 QPS 下，相比 vLLM，TTFT Mean 改善 24.0%，TTFT P95 改善 10.7%；在同一负载点下，相比 Company-X，TPOT Mean 仅下降 4.4%，TPOT P95 仅下降 3.1%。这些结果说明，当前模拟器不仅能够给出可用于排序的预测结果，而且确实能够在端到端系统中转化为可观的调度收益。

除端到端性能对比外，目前还完成了模拟器预测准确性分析，并对误差分布进行了统计。结果显示，在 Agent 3.1 倍工作负载下，Qwen2.5-7B 模型有 84.09% 的请求预测误差位于 30% 以内，Qwen3-30B-A3B 模型有 75.97% 的请求预测误差位于 30% 以内。该结果说明，当前在线模拟器对 Dense 模型已经具备较好的数值预测能力；对于带有动态专家路由特性的 MoE 模型，虽然绝对误差更大，但整体误差分布仍处于可接受范围，说明模拟器已经能够较稳定地反映候选实例之间的相对性能差异。

## 三、存在的问题及解决思路

当前存在的问题在于若干关键实验现象还缺少进一步的解释和系统化验证。首先，预测准确性测试虽然已经得到初步结论，但当前结果主要反映预测误差分布，对于误差来源本身还缺少分析。特别是在 MoE 模型场景下，误差略高于 Dense 模型，仍需进一步判断究竟是动态专家路由导致的建模偏差、状态同步滞后，还是实例内部执行波动带来的影响。针对这一问题，下一步需要结合批处理级别的预测日志，以及对批的执行时间和预测时间差异进行逐项比对，并分析误差究竟主要来自状态偏移还是预测建模本身的不准确。

在已有测试过程中观察到一个值得进一步研究的现象：在 7B 模型部署场景下，若误用原本为 30B MoE 模型构建的模拟器指导调度，端到端性能与使用正确 7B 模型模拟器时相比变化并不大。这一现象说明，在线调度所依赖的可能并不是预测值的绝对精确程度，而是模拟器是否仍能较稳定地保留各候选实例之间的相对优劣顺序。基于这一观察，猜想不同模型之间可能存在一定的预测迁移能力，即在部署模型变化时，已有模型的预测表仍有机会继续用于调度决策。针对这一问题，后续计划设计专门的预测迁移测试，对“误用模拟器但性能变化有限”这一观察进行可重复、可量化的验证，并进一步分析其成立条件与适用边界。

## 四、下一阶段的工作计划和研究内容

下一阶段的工作主要集中在实验测试和论文撰写两个方面。首先，预测准确性测试虽然已经得到初步结论，但对于误差来源本身还缺少分析，尤其是 MoE 模型场景下误差略高于 Dense 模型，仍需进一步判断究竟是动态专家路由导致的建模偏差还是状态同步之后或实例内部执行波动带来的影响。其次，已有实验中已经观察到“误用 30B MoE 模拟器指导 7B 模型调度时，端到端性能变化不大”的现象，这提示不同模型之间可能存在预测迁移能力，但该现象还缺少系统化验证，因此需要进一步设计预测迁移分析和 Spearman 秩相关分析。再次，可扩展性分析虽然已有实验设计思路，但不同实例规模下预测模块吞吐上界与系统服务能力上界之间的对照关系还未形成完整结果。除此之外，系统架构图、状态演进示意图和部分实验图表仍待补全，当前论文已经具备主体内容，但在论证闭环与展示完整性上还有进一步完善空间。

因此下一阶段工作的计划如下: 
第一，继续围绕预测准确性结果做深入分析，希望基于 Dense 与 MoE 两类模型的端到端测试结果，对误差来源进行分类排查，重点分析是否存在状态不同步导致的预测误差，以及分析批处理时长的预测准确性，并据此决定是否需要对预测特征或状态校准机制做针对性优化。这样尝试提高预测准确性并查看是否能够带来更大的性能提升。

第二，集中完成预测迁移测试及其排序一致性分析。该计划的动机来自当前已经观察到的现象，即在部分测试中误用不同模型的模拟器后，端到端性能并未发生显著退化。为验证这一现象是否具有普遍性，后续将系统比较“部署模型”和“预测表来源模型”不一致时的端到端表现，同时记录两种预测方式对候选实例的排序结果，计算 Top-1 选择一致率和 Spearman 秩相关系数。该部分工作的重点不只是补齐数值，更是验证在线调度是否主要依赖实例间相对排序而非绝对预测值，从而为预测表的复用和快速适配提供更有说服力的工程依据。

第三，完成在线开销与可扩展性测试，形成对系统工程可用性的最终论证。后续将采用逻辑实例映射物理实例的方式构造更大规模调度场景，统计不同实例规模下预测模块的处理吞吐，并与 8 实例集群满足 SLO 约束时的服务能力上界进行对照。若结果能够证明预测模块吞吐始终高于系统服务能力上界，则可以较有力地说明该方案在更大规模部署中不会先于系统本身成为瓶颈。

第四，推进论文与中期材料的统一整理。后续将同步补齐系统架构图、状态同步机制图、预测复用流程图及关键实验图表，建立结论 - 数据 - 图表”之间的一一对应关系，并对摘要、章节小结和结论部分进行统一收束。通过这一阶段的整理，目标是使论文从“系统已实现、实验已有初步结果”推进到“关键结论证据完整、章节表达自洽、具备提交与答辩准备条件”的状态。
